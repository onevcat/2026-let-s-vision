# Slide Overview（40 + 5）

## 0. 总体规划
- 正文：40 min
- Q&A：5 min
- 建议页数：24 页正文 + 2 页 Q&A 备援（总计 26 页）
- 分配原则：
  - 主观点上主页（标题 + 1 句结论 + 2~3 个证据点）
  - 重复/次要观点放页脚注释或讲者备注

## 2. Slide 逐页分配

### S01-S02 开场（3 min）
- S01 标题页：AI Agent 的道与术（结论先行）
- S02 30 秒总主张：道术双核 + 今天的输出（可执行框架）
- 注释（次要观点）：避免工具横评、避免空泛趋势预测。

### S03-S06 时代判断（6 min）
- S03 为什么回顾时代：为了定位当下并推断将来
- S04 三阶段机制图：Tab 补全 -> 提示-响应指挥 -> 长时少指令自主任务
- S05 四阶段工具图（辅助沟通）：补全/Chatbot/Editor/Code Agent
- S06 当下判断：2025/12~2026 初能力台阶上升，人开始成为局部瓶颈
- 注释（次要观点）：三阶段讲机制，四阶段讲产品形态，不冲突。

### S07-S08 现实证据（3 min）
- S07 三图三句话：PR、Issue/backlog、AI 高使用占比
- S08 规模背景（脱敏）：235 万行 / 650 模块 / 25,000 文件
- 注释（次要观点）：只讲趋势不讲绝对值；不展示可回推信息。

### S09-S14 道（10 min）
- S09 道-总览：三个约束（context、memory/skill、协作范式）
- S10 约束一：`context = 内存`（有限、紧张、需预算）
- S11 约束一延伸：上下文漂移、跨 session 断裂、决策丢失
- S12 约束二：个人/团队 memory + skill 才能形成复利
- S13 约束三：从“人指挥多 agent”走向“多 agent 自协作”
- S14 跨越点：人工监督 -> 无人职守（`webhook`/`cron`/event）
- 注释（次要观点）：agent 云端化趋势、OpenClaw 的“随时随地可达”启发。

### S15-S20 术（10 min）
- S15 术-总览：任务、上下文、质量循环
- S16 任务切分：交付物先定义，验收标准前置
- S17 上下文工程：可控注入源 + 无关历史隔离
- S18 harness 策略：不绑定单一 harness，允许并存与自建
- S19 工程价值重排（旧实践弱化）：提早优化更重要、测试维护成本下降、DRY 边际下降
- S20 工程价值重排（既有实践增强）：可观测性、TDD、benchmark、可验证循环
- 注释（次要观点）：低质量模型导致“错误学习”的长期成本。

### S21-S23 双案例（6 min）
- S21 高层案例：大规模仓库 subagent 协作图（分工与汇总）
- S22 具体案例：iOS 18 NavigationStack 假死排障闭环
- S23 案例总结：标准化排障 -> 资产沉淀 -> 可复用模板
- 注释（次要观点）：案例目的不是炫技，而是展示“流程可复制”。

### S24 收束（2 min）
- 三句话收束：定义问题、搭建系统、资产沉淀
- 行动号召：明天可落地的最小步骤（1 个模板 + 1 个验证循环）

### S25-S26 Q&A（5 min）
- S25 预置问题：安全权限、模型成本、无人职守治理
- S26 预置问题：memory/skill MVP、如何避免团队断层
- 注释（次要观点）：根据现场问题决定是否加讲“云端 agent 架构细节”。

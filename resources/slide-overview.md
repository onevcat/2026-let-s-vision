# Slide Overview（40 + 5）

## 0. 总体规划
- 正文：40 min
- Q&A：5 min
- 建议页数：33 页正文 + 1 页尾页（总计 34 页）
- 默认节奏：主叙事页 70~90 sec；过渡/收束页 40~60 sec
- 分配原则：
  - 主观点上主页（标题 + 1 句结论 + 2~3 个证据点）
  - 每页信息密度下降，增加“过渡页/对照页/案例拆页”
  - 重复或次要观点放讲者备注

## 2. Slide 逐页分配

### S01-S03 开场（3 min）
- S01 标题页：AI Agent 的道与术（Agentic Engineering > Vibe Coding）
- S02 讲者切入（轻量趣味）：从“桌前写代码”到“边炒菜边编程”，背后是 agent 可达性提升与效率跃迁
- S03 30 秒总主张：道术双核 + 今天输出（可执行框架）
- 注释（次要观点）：避免工具横评；避免空泛预测。

### S04-S07 时代判断（6 min）
- S04 为什么回顾时代：定位当下并推断将来
- S05 四阶段统一图：Tab 补全 -> Chatbot（提示-响应回路） -> AI Editor（仓内协作） -> Code Agent（长时少指令自主任务）
- S06 当下判断：2025/12~2026 初能力台阶上升，人开始成为局部瓶颈；工程师价值重心迁移到定义问题、设边界、做验收
- S07 整体判断：序章已经落下（科技 tick-tock + AI 能力超摩尔定律），世界在加速变化
- 注释（次要观点）：与智能体协作不是一次性学习，而是持续校准边界。

### S08-S10 现实证据（3 min）
- S08 规模背景（脱敏）：Swift + Objective-C / 235 万行 / 650 模块 / 25,000 文件
- S09 三图之一：PR 趋势（产出速度与质量形态变化）
- S10 三图之二&三：Issue/backlog 趋势 + AI 高使用占比趋势（先高手飞轮，再组织扩散）
- 注释（次要观点）：只讲趋势不讲绝对值；不展示可回推信息。

### S11-S19 道（10 min）
- S11 道-总览：三个约束（context、memory/skill、协作范式）+ 一个方向（to-human -> to-agent）
- S12 目标定义：人类要的是结果，不是软件操作过程
- S13 约束一：`context = 内存`（有限、紧张、需预算）
- S14 约束一延伸：上下文漂移、跨 session 断裂、决策丢失
- S15 约束二：个人/团队 memory + skill 形成复利（个人收益：减少重复沟通与重复踩坑）
- S16 公司观察：实践推广多是自下而上（高手经验 -> 团队共识），传统自上而下偏慢；少数高手与多数同事的效率断层需要治理
- S17 约束三：从“人指挥多 agent”走向“多 agent 自协作”（直接驱动因子：开发者注意力稀缺）
- S18 跨越点：人工监督 -> 无人职守（`webhook`/`cron`/event）+ 风险提示（极端自动化下人可能只保留责任承担角色）
- S19 战略落点：平台能力/权限审计/可观测性/流程自动化会成为竞争关键
- 注释（次要观点）：agent 云端化与 OpenClaw“随时随地可达”可作佐证。

### S20-S28 术（10 min）
- S20 术-总览：任务、上下文、质量循环
- S21 任务切分：可验收交付单元（PR/patch/demo/reply）
- S22 验收前置：测试/日志/回滚/风险列表
- S23 上下文工程：可控注入源（README/架构图/关键文件/约束清单）
- S24 注意力与污染：无关历史隔离；模型注意力常见分布（开头与最近更强，中段易衰减）
- S25 harness 策略：不绑定单一 harness，允许并存与自建
- S26 skill 沉淀收益：流程 skill 化可节省 token 与重复试错成本
- S27 工程价值重排（旧观点变化）：过早优化在部分场景反而更重要、测试维护成本下降、DRY 边际下降
- S28 工程价值重排（既有实践增强）：可观测性、TDD、benchmark、可验证循环
- 注释（次要观点）：低质量模型导致“错误学习”的长期成本。

### S29-S32 双案例（6 min）
- S29 高层案例-场景：大规模仓库中的 subagent 协作问题
- S30 高层案例-方案：分工（检索/改动/验证）+ 汇总决策
- S31 具体案例：iOS 18 NavigationStack 假死排障闭环
- S32 案例总结：标准化流程 -> 资产沉淀 -> 可复用模板
- 注释（次要观点）：案例目的不是炫技，而是展示“流程可复制”。

### S33 收束（2 min）
- 三句话收束：定义问题、搭建系统、资产沉淀
- 收束补句：软件开发与产品设计将持续从 to-human 走向 to-agent
- 收束补句：memory/skill 的构建本质是“渐进式披露（progressive disclosure）”实践
- 行动号召：明天最小落地（1 个模板 + 1 个验证循环）

### S34 尾页（Q&A 期间常驻）
- S34 Thank you：保留联系方式/二维码（可选）+ 引用资料（References，占位，后续补齐）
- 注释（次要观点）：Q&A 采用现场互动，不做预置问题 slide；视现场问题决定是否口头补充“云端 agent 架构细节”。
